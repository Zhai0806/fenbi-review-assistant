# 错因诊断 Skill

## 核心理念

**废除僵化的错误标签体系。** "计算失误""概念混淆"这类笼统标签对考生没有指导意义。改为 AI 生成具体描述（specific_error + countermeasure），直接告诉考生"你哪里理解错了"以及"下次怎么避免"。

## 降低误诊率的关键方法

**先推理后诊断，而非直接贴标签。** 诊断流程：

1. **先看正确答案** — 理解正确解法应该是什么
2. **再看考生选错的选项** — 分析选了它说明考生走了哪条错误路径
3. **结合用时 + 全站正确率** — 判断是知识盲点、计算失误、审题偏差还是时间压力

**常见错误路径对照表：**

| 现象 | 可能的真实原因 |
|------|--------------|
| 选错选项与正确选项含义相反 | 审题时忽略了否定词/提问方向（如问"不正确"选成正确的） |
| 选错选项是某个中间计算结果 | 公式步骤不完整，差最后一步 |
| 选错选项与正确选项是同类但不同对象 | 概念区分不清（如环比vs同比、增长率vs增长量） |
| 选错选项是题干中直接出现的数字 | 没有真正计算，直接抄数 |
| 用时极短（<10秒）+ 全站正确率不低 | 很可能是蒙的或没认真读题 |
| 用时正常但选了明显不相关的选项 | 知识点完全不会，凭感觉猜 |

## 诊断策略

**优先给具体原因，而非笼统分类。** 好例子 vs 坏例子：

| 坏（旧方式） | 好（新方式） |
|-------------|------------|
| 概念混淆 | 混淆了环比与同比的定义——题干给的是上月数据应套环比公式，但选了同比增长率 |
| 记忆盲区 | 对民法典第X条中"限制民事行为能力人的例外情况"记忆不清 |
| 审题不清 | 题目问"不正确"的选项，你选成了正确的——忽略了题干中的否定词 |
| 计算失误 | 基期计算第二步除法时，分子分母位置颠倒了——把现期当成了分母 |

## 系统提示词

你是公考行测辅导专家。深度诊断每道错题，先推理后判断，给出具体可操作的错因+对策。

诊断方法：
1. 先分析正确选项的解法是什么
2. 再看考生选错的选项内容，推断错误路径
3. 结合用时和全站正确率判断根本原因

模块判断参考（不应死板套用，应结合具体选项内容分析）：
- 常识/政治 → 通常为记忆问题
- 资料/数量 → 通常为计算或公式问题
- 言语/判断 → 通常为理解或审题问题
- 用时<10秒 → 可能为蒙题或放弃

置信度校准：如果错误路径有多种可能→降低 confidence；如果错选与正确选项差异明显→可提高 confidence。

输出 JSON：
```json
{
  "items": [
    {
      "specific_error": "把环比当成同比——题干给的是上月数据应套环比公式，但选了同比增长率",
      "countermeasure": "先圈出题干时间词确认比较基准——和上月比用环比，和去年同月比用同比",
      "explanation": "选了B说明他把增长量当成了增长率，看到增长率就直接套了同比公式",
      "confidence": 0.85
    }
  ]
}
```

### 字段说明

| 字段 | 用途 | 说明 |
|------|------|------|
| **specific_error** | **前端：错因** | 必须结合考生具体选错的选项内容来分析，说清错在哪个步骤、漏掉了哪个条件 |
| **countermeasure** | **前端：对策** | 一句可执行的具体对策（20-60字） |
| explanation | 前端辅助展示 | 简述考生的可能思维路径（20-80字） |
| confidence | 后台排序用 | 0-1，诊断依据充分→高；信息不足→低

## 整体复盘分析

整体分析细化到**模块下属题型**粒度（如 言语理解→逻辑填空），结合全站正确率区分：
- 全站正确率也低 → 题目本身偏难，不完全是你个人原因
- 全站正确率高但你做错了 → 真正的个人薄弱环节

融入认知科学方法：
- **认知负荷评估**：高用时低正确率的题型→优化策略或战略性跳过
- **元认知校准**：用时短但自信答对的题实际错了→判断不准确
- **测试效应**：错题重做时的提升空间评估
- **间隔重复**：按遗忘曲线安排复习节奏
