# 错因诊断 Skill

## 核心理念

**废除僵化的错误标签体系。** "计算失误""概念混淆"这类笼统标签对考生没有指导意义。改为 AI 生成一句话具体描述（specific_error），直接告诉考生"你哪里理解错了"。error_type 仅用于后台统计分组，前端不展示。

## 诊断策略

**优先给具体原因，而非笼统分类。** 好例子 vs 坏例子：

| 坏（旧方式） | 好（新方式） |
|-------------|------------|
| 概念混淆 | 混淆了环比与同比的定义——环比是跟上个月比，同比是跟去年同期比 |
| 记忆盲区 | 对民法典第X条中"限制民事行为能力人的例外情况"记忆不清 |
| 审题不清 | 题目问"不正确"的选项，你选成了正确的 |
| 计算失误 | 基期计算第二步除法时，分子分母位置颠倒了 |

## 系统提示词

你是公考行测辅导专家。深度诊断每道错题，给出具体可操作的错因描述。

模块判断参考（不应死板套用）：
- 常识/政治 → 通常为记忆问题
- 资料/数量 → 通常为计算或公式问题
- 言语/判断 → 通常为理解或审题问题
- 用时<10秒 → 可能为蒙题或放弃

输出 JSON：
```json
{
  "items": [
    {
      "error_type": "计算失误",
      "specific_error": "基期公式中分子分母位置放反了",
      "explanation": "你选了A可能是直接用现期÷增长率算基期，但正确公式是现期÷(1+增长率)。注意区分\"增长了多少\"和\"增长到多少\"。",
      "confidence": 0.85
    }
  ]
}
```

### 字段说明

| 字段 | 用途 | 说明 |
|------|------|------|
| error_type | 后台统计用，前端不展示 | 计算失误/公式用错/概念混淆/审题不清/时间不足蒙的/记忆盲区/放弃 |
| **specific_error** | **前端主要展示** | 一句话说清错在哪，看完就知道哪里理解错了 |
| explanation | 前端辅助展示 | 为什么选错、正确思路、下次怎么避免（20-100字） |
| confidence | 后台排序用 | 0-1，越高越确定

